# Contributing = a ZABAL Gamez submission

You do not need to build something big. Pick one small thing, add it, open a pull request. That pull request is a submission.

This is how ZABAL Gamez works: small moves, out loud, every day. A fix, a button, a line of copy, a test, a cleaner README - all of it counts.

## How to submit

1. Fork this repo (or any project in the lab: github.com/bettercallzaal).
2. Make one small, self-contained change.
3. Run `npm run build` so it stays green.
4. Open a pull request. Say what you changed in one line.

That is a submission. No minimum size.

## Good first additions to this repo

- a new theme line or `voicePhrases` entry in `lib/voice.ts`
- another issue in the daily-3 sequence (`lib/issues.ts`) or via the `/issues` manager
- a LinkedIn or Telegram variant alongside the Farcaster and X ones (`lib/variants.ts`)
- a new voice check in the score (`lib/score.ts`)
- a light theme toggle
- a keyboard shortcut in the command palette
- copy or accessibility fixes anywhere

## What "day one" looked like

The whole first day of this repo was built exactly this way: 13 small passes, each shipped live before the next. See [BUILD-LOG.md](./BUILD-LOG.md). Every pass there is a template for a submission.

## Voice

If you touch anything user-facing, keep the ZAO voice: lowercase-casual, lead with the build, real numbers, no emojis, no em dashes. See the "Voice" section in the README.
